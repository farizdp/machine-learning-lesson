2. Dataset: Training Set vs. Test Set

When you have a labelled dataset, you do not train the model on all of it. You split it into two separate parts:

🏋️ Training Set (~70–80% of data)
The model learns from this data. It sees the inputs and the correct outputs, and adjusts its internal parameters to get better at predicting the right answers.
🧪 Test Set (~20–30% of data)
The model has never seen this data during training. After training is complete, you use the test set to measure how well the model actually performs on new, unseen examples — simulating real-world use.
Analogy
Think of exam preparation. You study from a practice question book (training set). The real exam questions (test set) are different — you've never seen them before. Your exam score measures whether you truly learned the material, or just memorized the practice questions.

Why the split matters: a model could memorize all its training examples and score 100% on training — but fail completely on new data. Keeping the test set hidden gives you an honest measure of real performance.

⚠ Exam Alert
Know what each set is for: Training set = the model learns from it. Test set = measures real-world performance on unseen data. The test set must never be used during training.

3. Data Processing

Real-world data is messy. Before training, you must clean and transform it. There are three sub-steps:

① Data Cleansing

Fix problems in the raw data:

Filtering
Remove irrelevant or duplicate records (e.g. remove entries with obviously impossible values like age = -5).
Missing Value Handling
What to do when a field is blank: fill with the average (imputation), or remove the row entirely.
Abnormal / Error Value Handling
Detect and fix outliers — values so extreme they are probably errors (e.g. a house price of $1 in a normal neighbourhood).
Merging & Data Consolidation
Combine data from multiple sources into one consistent dataset (e.g. merging customer records from two different databases).

② Data Standardization (Conversion)

Models work best when all numerical features are on a similar scale. If one feature ranges from 0–1 (e.g. a probability) and another ranges from 0–1,000,000 (e.g. income), the large-scale feature will dominate unfairly.

Two common techniques:

Min-Max Normalization
Rescale every value to fit between 0 and 1. Formula: (x − min) / (max − min)
Z-score Standardization
Rescale so the data has mean = 0 and standard deviation = 1. Formula: (x − mean) / std

③ Dimensionality Reduction

Sometimes your data has hundreds or thousands of features (columns). Many are redundant or irrelevant. Dimensionality reduction compresses the data into fewer features while keeping the important information.

Why it matters: fewer features = faster training, less overfitting, easier to visualize.

Key technique: PCA (Principal Component Analysis) — finds the directions of maximum variation and projects the data onto them.

⚠ Exam Alert
Data processing has three stages: Cleansing (fix bad data), Standardization (rescale to same range), Dimensionality Reduction (fewer features, same information). Know what each stage solves.

4. Feature Selection

A feature is one input variable — one column in your dataset. For a house price model: size, age, number of bedrooms, distance to school — each of these is a feature.

Not all features are useful. Some are irrelevant noise. Feature selection picks the best subset to use for training. Three methods:

① Filter Method
Evaluate each feature independently, without involving any model. Score each feature by how well it correlates with the target (e.g. using statistical tests). Fast, but ignores how features interact with each other.

Analogy: Screening job applicants by looking at their CV score alone, before any interview.
② Wrapper Method
Use the actual model to test different subsets of features. Try all combinations (or smart subsets), train a model on each, and pick the subset that performs best. Slow but accurate.

Analogy: Interviewing every possible team combination to find the best group chemistry.
③ Embedded Method
Feature selection happens during model training itself — the model learns which features are important as part of its optimization. Efficient: one step instead of two.

Example: Decision trees and LASSO regression naturally ignore irrelevant features during training.
⚠ Exam Alert
Know all three: Filter = independent of model (fastest), Wrapper = tests feature subsets using a model (most accurate, slowest), Embedded = happens inside model training (balanced). A common question gives a scenario and asks which method is being used.

5. What Makes a Good Model?

A model isn't just judged by its accuracy score. Three qualities define a truly good model:

🌍 Generalization
Performs well on new, unseen data — not just the training data. This is the most important quality. A model that memorizes training examples but fails on new ones is useless in production.
💡 Explainability
Humans can understand why the model made a decision. Critical in high-stakes domains: a bank must explain why it denied a loan; a doctor must understand why an AI flagged a tumour. Not all models are equally explainable — decision trees are very explainable; deep neural networks are not.
⚡ Prediction Speed
Makes predictions fast enough for real-world use. A model that takes 30 seconds to classify one image is useless for a real-time security camera. Speed matters most at deployment, not just during training.

6. Overfitting, Underfitting, and the Bias-Variance Trade-off

The most common failure mode in ML is building a model that doesn't generalize. There are two opposite ways this happens:

😴
Underfitting
Model too simple. Doesn't learn the patterns even from training data. Bad on training AND test sets.
High Bias
Low Variance
✅
Just Right
Model captures the real pattern. Good on both training AND test sets.
Low Bias
Low Variance
🧠
Overfitting
Model memorizes training data, including noise. Great on training, poor on test set.
Low Bias
High Variance
Analogy
Imagine studying for an exam:
  • Underfitting: You barely studied. You can't even answer the practice questions.
  • Overfitting: You memorized every single practice question word-for-word. On the real exam — with different wording — you fail completely.
  • Just right: You understood the concepts. You can answer practice and new exam questions.

Bias and Variance — The Root Cause

Bias — "wrong assumptions"
Error from the model being too simple and making wrong assumptions about the data. High bias → underfitting. The model consistently gets things wrong in the same way.
Variance — "too sensitive"
Error from the model being too complex and reacting to noise in the training data. High variance → overfitting. Small changes in training data lead to wildly different models.
⚠ Exam Alert
The bias-variance trade-off:
High Bias = Underfitting (too simple, wrong on everything)
High Variance = Overfitting (too complex, memorizes noise)
Good models have low bias AND low variance — they generalize well.

How to Fix Overfitting

  • Get more training data — the most effective fix
  • Simplify the model — fewer parameters, shallower tree depth
  • Regularization — penalize model complexity (covered in deep learning lessons)
  • Dropout — randomly disable neurons during training (for neural networks)
  • Early stopping — stop training before the model starts memorizing noise
