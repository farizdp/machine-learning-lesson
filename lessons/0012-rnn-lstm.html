Recurrent Neural Networks & LSTM

How neural networks handle sequences — text, audio, time series — by remembering the past.

1. Why Standard Networks Struggle with Sequences

CNNs and MLPs process a fixed-size input and produce a fixed-size output. But many real tasks are sequential:

  • Translating a sentence (variable-length input → variable-length output)
  • Predicting the next word in "The cat sat on the ___"
  • Classifying the sentiment of a review sentence
  • Predicting tomorrow's stock price from the past 30 days

The key challenge: word order matters, and the meaning of each word depends on what came before it. A plain MLP treats every input as independent — it has no memory of what it just processed.

Analogy
Reading a sentence one word at a time, you carry a mental summary of what you've read so far. Each new word updates that summary. A standard network reads all words at once and can't build this running context. An RNN does exactly what you do — it keeps a "memory note" that gets updated at every step.

2. Recurrent Neural Networks (RNN)

An RNN processes one element of the sequence at a time. At each step it produces a hidden state — a vector that summarises everything seen so far — and passes it to the next step.

Each cell receives the current word xₜ and the previous hidden state hₜ₋₁, and outputs a new hidden state hₜ. The same cell (same weights) is reused at every step.

⚠ Exam Alert
The RNN hidden state update formula: hₜ = tanh(Wₓ · xₜ + Wₕ · hₜ₋₁ + b). Two weight matrices (Wₓ for input, Wₕ for previous hidden state), shared across all time steps. This weight sharing is what makes RNNs feasible for variable-length sequences.

3. The Vanishing Gradient Problem in RNNs

You saw in Lesson 09 that deep networks suffer from vanishing gradients — gradients shrink as they travel backward through layers. In RNNs the same problem appears in the time dimension: to learn that "The" at position 0 influences the output at position 50, the gradient must flow backward through 50 time steps, shrinking with each one.

The result: a plain RNN essentially has short-term memory only. It can't learn that a word from 50 steps ago is relevant to the current output. This is the core motivation for LSTM.

4. Long Short-Term Memory (LSTM)

LSTM was designed specifically to fix the vanishing gradient problem and give the network a way to selectively remember or forget information over long sequences. Instead of just a hidden state hₜ, an LSTM cell maintains two pieces of memory:

Cell State (Cₜ)
The long-term memory. It flows along the sequence with only minor, controlled modifications — a "conveyor belt" that preserves information across many steps. Gradients can flow through it without vanishing.
Hidden State (hₜ)
The short-term memory / output. Used for predictions and passed to the next step, just like in a plain RNN. Derived from the cell state at each step.
Analogy
Think of the cell state as a notebook and the hidden state as your working memory. Gates are like decisions you make at each moment: "Should I write this into my notebook? Should I erase an old note? What should I tell the next person from my notebook right now?"

The Four Gates

LSTM controls the cell state using three gates, each producing a value between 0 and 1 (via sigmoid: 0 = "block everything", 1 = "let everything through"), plus one input transformation:

The cell state (orange highway) flows top to bottom. Each gate decides how much information to pass or block.

Forget Gate (fₜ)
Decides what to erase from the long-term cell state. Output near 0: forget this information. Output near 1: keep it. Example: when starting a new sentence, the model should forget the subject of the previous sentence.
Input Gate (iₜ) + Candidate (C̃ₜ)
Works in two parts: the input gate decides which new information to write, and the candidate creates the actual new values to write. Together: new cell state Cₜ = fₜ × Cₜ₋₁ + iₜ × C̃ₜ.
Output Gate (oₜ)
Decides what part of the cell state to expose as the output (hidden state hₜ). The cell state is passed through tanh to squash values, then filtered by the output gate: hₜ = oₜ × tanh(Cₜ).
⚠ Exam Alert
LSTM has three gates: Forget Gate (erase old memory), Input Gate (write new memory), Output Gate (expose memory as output). It also has a cell state (long-term) and a hidden state (short-term / output). The cell state solves the vanishing gradient problem by allowing gradients to flow backward without shrinking.

5. RNN vs LSTM — When to Use Which

Property Plain RNN LSTM
Long-range dependencies Poor — gradient vanishes Good — cell state highway
Complexity / parameters Low ~4× more than plain RNN
Training speed Faster Slower
Typical use Short sequences, simple tasks Translation, speech, long text

6. Sequence-to-Sequence (Seq2Seq)

Seq2Seq is an architecture that uses two RNNs (or LSTMs) connected in series to map a variable-length input sequence to a variable-length output sequence. Classic use case: machine translation.

Encoder reads "The cat sat down" → compresses into a context vector C → Decoder generates the French translation one word at a time.

Encoder
An LSTM that reads the entire input sequence and compresses it into a fixed-size context vector (the final hidden state). All information about the input must fit in this vector.
Context Vector
The bottleneck: a fixed-size summary of the whole input. The encoder's final hidden state. Passed to the decoder as its initial hidden state.
Decoder
Another LSTM that receives the context vector and generates the output sequence one token at a time. Each output token is fed back as the next input, until a special <END> token is produced.
Limitation that motivated Transformers
The fixed-size context vector is a bottleneck — for long sentences, it can't hold all the information. This limitation led to the Attention Mechanism (Lesson 13), which lets the decoder look back at all encoder hidden states, not just the last one.
⚠ Exam Alert
Seq2Seq = Encoder LSTM + Context Vector + Decoder LSTM. Used for translation, summarisation, and speech recognition. The bottleneck context vector limitation led directly to the invention of attention and, later, the Transformer.
