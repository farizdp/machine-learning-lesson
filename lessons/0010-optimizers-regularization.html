Optimizers & Regularization

Train faster with smarter optimizers — then prevent your model from memorizing noise.

1. Why Plain Gradient Descent Isn't Enough

In Lesson 06 you learned about gradient descent: compute the gradient, step in the downhill direction, repeat. Simple — but in practice it has problems:

  • Slow convergence — the same learning rate is used for every weight, even if some weights need bigger or smaller steps
  • Oscillation — the loss can bounce side-to-side in narrow valleys instead of heading straight toward the minimum
  • Local minima and saddle points — the gradient points to zero but you haven't found the best solution

Optimizers are improved algorithms that modify how gradient descent updates weights to avoid these problems.

2. The Optimizer Family

① Momentum

Momentum adds a "memory" of past gradients. Instead of taking a step based only on the current gradient, it accumulates a velocity vector — like a ball rolling downhill that builds up speed.

Analogy
A ball rolling down a hill doesn't stop and re-evaluate every millisecond. It builds momentum in the downhill direction and rolls past small bumps. Plain gradient descent is like a short-sighted person taking tiny cautious steps — Momentum is the ball that keeps rolling.
How it works
Maintain a velocity v. Each step: v ← β·v + (1−β)·∇L, then update w ← w − η·v. The hyperparameter β (typically 0.9) controls how much past gradients influence the current step.
Advantages
  • Accelerates learning in the consistent gradient direction
  • Dampens oscillations in narrow valleys (the side-to-side bouncing cancels out)
  • Can escape shallow local minima
Disadvantages
  • Adds a new hyperparameter β to tune
  • Can overshoot the minimum if β is too large
  • Still uses one global learning rate for all weights

② AdaGrad (Adaptive Gradient)

AdaGrad adapts the learning rate per weight — weights that have received large gradients in the past get a smaller learning rate; weights that have received small gradients get a larger one.

Key idea
Accumulate the sum of squared past gradients for each weight. Divide the learning rate by the square root of this sum. Frequently updated weights slow down; rarely updated weights stay fast.
Best for
Sparse data (like NLP, where most words appear rarely). Each word's embedding gets its own appropriate learning rate.
Weakness
The accumulated squared gradients only grow — the learning rate shrinks monotonically and can reach near-zero long before training is done. The model stops learning too early. RMSProp fixes this.

③ RMSProp (Root Mean Square Propagation)

RMSProp fixes AdaGrad's "learning rate dies" problem by using an exponential moving average of squared gradients instead of a running total. Old gradients fade out; only recent gradient magnitude matters.

Key idea
v ← β·v + (1−β)·(∇L)², then w ← w − η/√v · ∇L. The β parameter controls how quickly old gradients are forgotten (typically 0.9).
Advantage over AdaGrad
The learning rate never permanently vanishes — it adapts to recent gradient history rather than all-time history.
Weakness
Does not have a momentum term — does not build speed in consistent directions. Adam combines RMSProp with Momentum to get both benefits.

④ Adam (Adaptive Moment Estimation)

Adam combines the best of both worlds: Momentum (builds speed in consistent directions) + RMSProp (per-weight adaptive learning rates). It is the most widely used optimizer in deep learning today.

What it tracks
Two moving averages per weight: m (first moment = mean of gradients, like Momentum) and v (second moment = mean of squared gradients, like RMSProp). Both are bias-corrected at the start of training to avoid cold-start under-estimates.
Why it dominates
Works well out of the box across a wide range of problems. Rarely needs learning rate tuning. Handles sparse gradients and noisy data well. Default in PyTorch, TensorFlow, and MindSpore.
Hyperparameters
Learning rate η (default 0.001), β₁ (default 0.9), β₂ (default 0.999), ε (small constant for numerical stability, default 1e-8).

⚠ Exam Alert
Know the progression: Momentum adds speed. AdaGrad adapts per-weight LR (but LR permanently decays). RMSProp fixes AdaGrad's decay with exponential moving average. Adam = Momentum + RMSProp = the default optimizer. AdaGrad's weakness (permanent LR decay) is a common exam question.

3. Regularization — Fighting Overfitting

You learned in Lesson 05 that overfitting means the model memorizes training data but fails on new data. Regularization is the set of techniques that force the model to stay simpler and generalize better.

① L1 Regularization (Lasso)

Adds the sum of absolute values of all weights to the loss function: Loss_total = Loss + λ·Σ|w|

Effect
Pushes many weights exactly to zero. This produces a sparse model — most weights are inactive, only a few matter.
Best for
Feature selection — L1 automatically discards irrelevant features by zeroing their weights. Useful when you suspect only a few features are actually important.

② L2 Regularization (Ridge / Weight Decay)

Adds the sum of squared values of all weights to the loss: Loss_total = Loss + λ·Σw²

Effect
Pushes all weights toward zero but rarely exactly to zero — produces small, spread-out weights. Prevents any single weight from becoming dominant.
Best for
General overfitting prevention. The most common regularization in neural networks (also called weight decay in this context). More stable than L1 when features are correlated.
L1 (Lasso)
Penalty: λ·Σ|w|
Result: sparse — many weights = 0
Use: automatic feature selection
L2 (Ridge / Weight Decay)
Penalty: λ·Σw²
Result: small weights spread across all features

λ (lambda) is the regularization strength hyperparameter — higher λ = stronger penalty = simpler model
⚠ Exam Alert
L1 → sparse weights (many exactly zero) → feature selection. L2 → small weights (spread out, rarely zero) → general regularization. Both work by adding a penalty to the loss that discourages large weights.

③ Early Stopping

During training, monitor the model's performance on a validation set (a small portion of data held out from training). Stop training when validation loss starts to increase even though training loss is still decreasing — that's the moment overfitting begins.

Stop when validation loss bottoms out — before the overfitting gap widens

Advantage
No extra hyperparameters. Free regularization — just watch the validation curve and stop at the right moment.
In practice
Use patience: stop only if validation loss has not improved for N consecutive epochs (e.g. N=10), to avoid stopping too early on a temporary spike.

④ Dropout

During each training step, randomly deactivate (drop) a fraction of neurons — typically 20–50% — so they contribute nothing to that forward pass. Different neurons are dropped each time.

Training (Dropout ON)
Grey ✕ = dropped this step
Inference (Dropout OFF)
All neurons active — outputs scaled
Why it prevents overfitting
Neurons can't rely on any specific other neuron — they must learn robust, independent features. The network effectively trains many different sub-networks simultaneously and averages them.
At inference time
Dropout is turned off. All neurons are active. To compensate for the increased activation magnitude, outputs are scaled down by the dropout rate.
Typical dropout rates
0.2–0.5 for hidden layers. Too high = underfitting (too much information lost). Too low = no regularization effect.
⚠ Exam Alert
Four regularization techniques: L1 (sparse, feature selection), L2 (small weights, general), Early Stopping (monitor validation loss, stop before overfitting), Dropout (randomly drop neurons during training only — OFF at inference). Dropout is only applied during training — a common exam trap.
