Activation Functions & How Neural Networks Learn

The functions that give networks their power — and the algorithm that teaches them.

1. Common Activation Functions

Every neuron applies an activation function after its weighted sum. In Lesson 08 you learned why nonlinearity is essential — now let's meet the specific functions used in practice. The exam expects you to know all seven.

① Step Function

What it does
Outputs 1 if the input exceeds a threshold, 0 otherwise. Binary on/off — mimics the original biological neuron "fires or doesn't fire."
Problem
The gradient is zero everywhere except exactly at the threshold — so gradient descent can't learn through it. Not used in modern deep learning.

② Sigmoid (Logistic)

σ(z) = 1 / (1 + e−z) — output always between 0 and 1

Use
Output layer of binary classifiers — the output is a probability. Already covered in Logistic Regression.
Problem
Vanishing gradient — for very large or very small inputs, the curve is nearly flat (gradient ≈ 0). Gradients shrink to near-zero as they flow back through many layers, stalling learning.

③ Tanh (Hyperbolic Tangent)

Output range: (−1, +1) — zero-centered

Advantage over Sigmoid
Output is zero-centered (−1 to +1 instead of 0 to 1). This makes gradient updates more symmetric and often speeds up training.
Problem
Still suffers from vanishing gradients at the extremes. Less common in hidden layers today.

④ ReLU (Rectified Linear Unit)

f(z) = max(0, z) — zero below 0, linear above 0

Why it dominates
Gradient is exactly 1 for all positive inputs — no vanishing gradient on the positive side. Extremely fast to compute. The default activation function for hidden layers today.
Problem — Dying ReLU
If a neuron's input is always negative, it always outputs 0, and its gradient is always 0 — it permanently stops learning ("dies"). Solution: Leaky ReLU.

⑤ Leaky ReLU

What it does
Same as ReLU for positive inputs. For negative inputs, instead of outputting exactly 0, it outputs a small negative value: f(z) = max(0.01z, z).
Why it fixes Dying ReLU
The tiny slope (0.01) on the negative side means the gradient is never exactly zero — neurons can always recover and keep learning.

⑥ Swish

Formula
f(z) = z · σ(z) — the input multiplied by its own sigmoid. Proposed by Google in 2017.
Why it's used
Smooth everywhere (unlike ReLU's sharp corner at 0), allows small negative values through, and empirically outperforms ReLU on some deep networks. Used in models like EfficientNet and some Transformer variants.

⑦ Softmax

What it does
Takes a vector of raw scores (one per class) and converts them into probabilities that sum to 1. softmax(zᵢ) = ezᵢ / Σezⱼ
Where it lives
Output layer of multi-class classifiers only. For a network classifying 10 digits (0–9), the output layer has 10 neurons, Softmax converts their scores into 10 probabilities. The highest probability wins.
Function Output range Used where Key weakness
Step {0, 1} Historical only Zero gradient everywhere
Sigmoid (0, 1) Binary output layer Vanishing gradient
Tanh (−1, +1) Hidden layers (older) Vanishing gradient
ReLU ★ [0, ∞) Hidden layers (default) Dying ReLU
Leaky ReLU (−∞, ∞) Hidden layers Extra hyperparameter (slope)
Swish (−∞, ∞) Deep hidden layers Slightly more compute
Softmax (0,1) sums to 1 Multi-class output only Not for hidden layers
⚠ Exam Alert
ReLU = default for hidden layers. Sigmoid = binary output layer. Softmax = multi-class output layer (probabilities sum to 1). Tanh = zero-centered, but still vanishing gradient. Leaky ReLU = fixes Dying ReLU by allowing a small negative slope.

2. Loss Functions

A loss function (also called cost function) measures how wrong the model's prediction is compared to the correct answer. Training is literally the process of minimizing this number.

Mean Squared Error (MSE) — for Regression
MSE = (1/n) Σ(ŷᵢ − yᵢ)²
Squares the error so large mistakes are punished heavily. Always non-negative. Output: a single number representing average squared error over all training examples.
Cross-Entropy Loss — for Classification
Measures the difference between the predicted probability distribution and the true label. When the model assigns high probability to the correct class, loss is low; when it's confident about the wrong class, loss is very high.
Binary cross-entropy for 2 classes; categorical cross-entropy for multiple classes (used with Softmax).
⚠ Exam Alert
MSE = regression tasks. Cross-Entropy = classification tasks. The choice of loss function depends on the task type, not the model architecture.

3. How a Neural Network Learns

Training a neural network is a two-pass process repeated thousands of times:

① Forward Pass
Input flows left→right through all layers. Each layer computes z = Wx + b, then a = f(z). Final layer produces prediction ŷ.
② Compute Loss
Compare ŷ (predicted) to y (true label) using the loss function. Get a single number: how wrong were we?
③ Backward Pass
Compute how much each weight contributed to the loss. Flow gradients right→left through all layers using the chain rule. This is backpropagation.
④ Update Weights
Adjust every weight by a small step in the direction that reduces loss: w ← w − η · ∂L/∂w. (η is the learning rate.)
↺ Repeat for every batch until loss converges
Analogy — Learning to Throw a Basketball
You throw (forward pass), see where the ball lands (compute loss). You work backwards — was it the wrist angle? The elbow? The jump? (backpropagation). You adjust each joint slightly (update weights). Throw again. Each attempt, you get a little closer to the basket.

4. Backpropagation & the Chain Rule

Backpropagation (backprop) is the algorithm that computes gradients efficiently through a deep network. It applies the chain rule from calculus.

The Chain Rule in plain English
If A affects B and B affects C, then the total effect of A on C is the product of both effects:
∂C/∂A = (∂C/∂B) × (∂B/∂A)

In a neural network: output loss depends on layer 3, which depends on layer 2, which depends on layer 1. The chain rule lets you compute "how much did layer 1's weights contribute to the final loss?" by multiplying gradients along the path.

Backprop works by starting at the loss, computing the gradient with respect to the output layer's weights, then chaining backwards through each layer until every weight in the network has a gradient. Then gradient descent uses those gradients to update all weights in one step.

⚠ Exam Alert
Backpropagation = efficient application of the chain rule to compute gradients for all weights in a network. It flows backward (right to left), while the forward pass flows left to right. Without backprop, training deep networks would be computationally impossible.

5. Vanishing & Exploding Gradients

As gradients are chained backwards through many layers, they can become problematic in two opposite ways:

📉 Vanishing Gradients
Gradients are multiplied together as they flow backward. If each layer's gradient is a small number (e.g. the flat ends of Sigmoid/Tanh), multiplying many small numbers together gives a number close to zero. Early layers receive near-zero gradients and learn almost nothing — the network effectively only trains its last few layers.

Common cause: Sigmoid or Tanh activations in deep networks.
Solutions: Use ReLU activations; use Batch Normalization; use skip connections (ResNet).
📈 Exploding Gradients
The opposite problem: gradients grow exponentially as they flow backward, eventually becoming so large that weight updates blow up (weights become NaN or infinity). More common in RNNs.

Solutions: Gradient clipping — cap gradient magnitude at a maximum value before updating. Also: careful weight initialization (Xavier, He initialization).
📉 Vanishing
gradient × gradient × … ≈ 0
Early layers freeze. Network learns poorly.
Fix: ReLU · Batch Norm · ResNet skip connections
📈 Exploding
gradient × gradient × … = ∞
Weights blow up. Training diverges.
Fix: Gradient clipping · Xavier/He init
⚠ Exam Alert
Vanishing gradient: caused by activations with small derivatives (Sigmoid, Tanh); fix with ReLU. Exploding gradient: gradients grow out of control; fix with gradient clipping. Both are problems of very deep networks. ReLU's constant gradient of 1 for positive inputs is the main reason it replaced Sigmoid in hidden layers.
