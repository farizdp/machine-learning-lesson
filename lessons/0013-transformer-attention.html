Transformer Architecture & Attention

The architecture behind GPT, BERT, and every modern large language model — and why it replaced RNNs overnight.

1. The Problem Attention Was Built to Solve

In Lesson 12 you saw that Seq2Seq has a bottleneck: the entire input sentence must be compressed into a single fixed-size context vector. For long sentences, information gets lost.

Analogy
Imagine translating a 200-word paragraph. The Seq2Seq approach requires you to read the whole thing, write a single one-paragraph summary on a notecard, then translate from only that notecard — discarding the original. Attention lets you keep the original and look back at any part of it whenever you need it while translating.

Attention fixes this by letting the decoder, at each output step, look at all encoder hidden states — not just the last one — and decide which input words are most relevant right now.

2. The Attention Mechanism

Attention computes a weighted sum of the encoder's hidden states, where the weights reflect how relevant each encoder position is to the current decoder step.

Query (Q)
What the decoder is currently looking for. "I'm generating the French word for 'cat' — what encoder positions are relevant?" One vector per decoder step.
Key (K)
What each encoder position offers. "This position encoded 'cat', this one encoded 'sat'…" One vector per encoder position.
Value (V)
The actual content to retrieve from each encoder position. Once attention scores tell us how relevant each position is, we take a weighted sum of its Value vectors.
Q / K / V as a library search
Query: what you type in the search bar. Keys: book titles and tags in the catalogue. Values: the actual book content. Attention computes how well each key matches your query, then blends the corresponding values proportionally.

When generating "le chat", the decoder's Query matches heavily against "cat" (score 0.88). The context vector is dominated by that encoder position's Value.

3. Self-Attention

The Transformer took attention one step further: instead of attention between encoder and decoder, it introduced self-attention — each position in a sequence attends to all other positions in the same sequence to build richer representations.

Why self-attention matters
In the sentence "The animal didn't cross the street because it was too tired," what does "it" refer to? A human knows it refers to "animal." Self-attention lets the model figure this out: when encoding "it", the model looks at all other words and assigns high attention weight to "animal" — resolving the reference without any special coreference module.

When encoding "it", self-attention assigns a high weight to "animal" — resolving the pronoun reference automatically.

⚠ Exam Alert
Self-attention formula: Attention(Q, K, V) = softmax(Q·Kᵀ / √dₖ) · V. The √dₖ is a scaling factor that prevents dot products from becoming too large (which would cause softmax to saturate). Q, K, V are all derived from the same sequence by multiplying with learned weight matrices.

4. Multi-Head Attention

Running self-attention once captures one kind of relationship. Multi-head attention runs self-attention h times in parallel (the "heads"), each with its own Q/K/V weight matrices. The results are concatenated and projected back down.

Each head learns a different type of relationship (syntax, coreference, long-range dependencies…). The outputs are concatenated and projected back to the original dimension.

5. Positional Encoding

Self-attention looks at all positions simultaneously — it has no built-in concept of order. "The cat sat" and "sat cat The" would produce the same attention scores. To fix this, positional encodings are added to each word's embedding before it enters the Transformer.

How it works
A vector of sine and cosine waves at different frequencies is generated for each position and added to the word embedding. Nearby positions produce similar encodings; distant positions produce very different ones. The model can learn to use this positional signal to understand order.
Learned vs Fixed
The original Transformer uses fixed sinusoidal encoding. Modern models (GPT, BERT) often use learned positional embeddings instead — just another set of parameters trained from data.
⚠ Exam Alert
Positional encoding is added to word embeddings before the first Transformer layer. Without it, the model is permutation-invariant — it cannot distinguish "dog bites man" from "man bites dog".

6. The Transformer Architecture

The original Transformer (Vaswani et al., "Attention Is All You Need", 2017) uses an encoder stack and a decoder stack, each made of repeated identical layers.

The encoder builds rich representations of every input token. The decoder generates output tokens one at a time, attending to both its own past outputs and all encoder positions.

Add & Norm (Residual Connection + Layer Norm)
After each sub-layer, the input is added back to the output (residual/skip connection, like ResNet), then layer normalisation is applied. This lets gradients flow directly and stabilises training.
Masked Self-Attention (decoder only)
During training, the decoder cannot peek at future tokens. A mask blocks attention to positions after the current one — ensuring the model learns to predict each token from only previous tokens.
Cross-Attention (decoder only)
The decoder's Query comes from the decoder itself; Keys and Values come from the encoder output. This is how the decoder reads the input while generating output — the modern version of the Seq2Seq attention from Lesson 12.
⚠ Exam Alert
The original Transformer has: Encoder (Multi-Head Self-Attention → FFN, repeated N times) and Decoder (Masked Self-Attention → Cross-Attention → FFN, repeated N times). Key advantages over RNN: parallelisable (all positions processed simultaneously during training) and no vanishing gradient over sequence length.

7. GPT-1 and BERT — Two Ways to Use the Transformer

Modern models don't always use both encoder and decoder. They specialise:

Property BERT (Encoder-only) GPT-1 (Decoder-only)
Architecture Encoder stack only Decoder stack only (masked)
Pre-training task Masked Language Modelling — predict randomly masked words Causal Language Modelling — predict the next word
Attention direction Bidirectional — sees left AND right context Left-to-right only — cannot see future tokens
Best for Understanding tasks: classification, NER, Q&A Generation tasks: text completion, dialogue, code
Lineage Google, 2018. Fine-tuned on many NLP tasks. OpenAI, 2018. Led to GPT-2, GPT-3, GPT-4.
⚠ Exam Alert
BERT = Encoder-only, Bidirectional (sees full context, good for understanding). GPT = Decoder-only, Unidirectional (left-to-right, good for generation). Both are pre-trained on large text corpora and then fine-tuned for downstream tasks.

8. Mixture of Experts (MoE)

A standard Transformer's feed-forward layer activates all parameters for every input token. Mixture of Experts (MoE) replaces that layer with many expert sub-networks, and a router that selects only a small number of them (e.g. top-2 out of 64) for each token.

Why MoE?
Increases model capacity (more total parameters) without proportionally increasing computation — because only a fraction of experts run per token. A 100B-parameter MoE model might only compute as much as a 10B dense model per token.
Used in
DeepSeek, Mixtral, and GPT-4 reportedly use MoE. It's a major reason modern frontier models can be so large yet remain feasible to run.
⚠ Exam Alert
MoE key idea: many expert FFN layers + a router that activates only top-k experts per token. Result: large total parameters, small active parameters per token. This trades more memory for less computation.
