AI Application Development Process

How a real AI project goes from a problem statement to a deployed application — the six stages every developer follows.

1. The Six-Stage AI Development Pipeline

Building an AI model is not just about writing a neural network. There are six stages, and skipping any of them causes the project to fail in production. The Huawei HCIA-AI syllabus explicitly defines this pipeline:

AI development is a loop, not a straight line. Feedback from deployment drives improvements in data and the model.

⚠ Exam Alert
Memorise the six stages in order: Requirement Analysis → Environment Setup → Data Preparation → Network Building → Model Training → Application Development. The exam often asks you to identify which stage a described activity belongs to.

2. Each Stage Explained

① Requirement Analysis
What: Define the problem clearly before writing a single line of code. Is it classification, regression, or detection? What counts as acceptable accuracy? What are the latency and memory constraints?
Outputs: Task type, success metrics (accuracy / F1 / latency), data requirements, hardware targets.
② Environment Setup
What: Install and configure the frameworks and hardware that will run your model.
Huawei context: MindSpore installed on Ascend hardware, or using ModelArts (Huawei's cloud AI platform) which provides a managed environment where frameworks, drivers, and compute are pre-configured. Saves days of environment debugging.
③ Data Preparation
What: Collect, clean, label, split, and augment data — typically 60–70% of the total project time.
Sub-steps (from Lesson 05): Data cleansing → Standardisation → Dimensionality reduction → Feature selection → Split into train / validation / test sets.
Key principle: Garbage in, garbage out. No model can compensate for poor quality data.
④ Network Building
What: Choose an architecture and implement it in MindSpore (or another framework). Decisions: number of layers, layer types, activation functions, whether to start from scratch or use a pre-trained model.
Example: For digit recognition, start with LeNet-5 (Lesson 14). For a new image task with limited data, use a pre-trained ResNet and fine-tune (see Lesson 16).
⑤ Model Training
What: Feed data through the network, compute loss, run backpropagation, update weights — repeated across many epochs.
Key decisions: Loss function, optimiser (usually Adam), learning rate schedule, batch size, number of epochs, when to apply regularisation (Dropout, L2).
Monitor: Training loss and validation accuracy at each epoch. Stop when validation accuracy stops improving (early stopping).
⑥ Application Development
What: Package the trained model for real-world use. Export it, wrap it in an API or app, deploy it to a server, edge device, or the cloud.
MindSpore tools:
  • mindspore.save_checkpoint() — save trained weights
  • mindspore.export() — export to ONNX, MindIR, or AIR format
  • MindSpore Serving — production inference server for Ascend/GPU
  • MindSpore Lite — compressed model for mobile / edge devices

3. Worked Example: Digit Recognition with LeNet on ModelArts

Let's walk the full six stages for a concrete project — building a handwritten digit classifier using LeNet-5 on Huawei's ModelArts platform.

Stage What happens in this project
① Requirement Classify 28×28 greyscale images into digits 0–9. Target: >99% accuracy on MNIST test set. Inference must run in under 10ms.
② Environment Create a ModelArts training job — select MindSpore 2.x, Ascend 910 instance. No local setup needed.
③ Data Download MNIST (60,000 train / 10,000 test images). Normalise pixel values to [0, 1]. Upload to OBS (Huawei Object Storage). No augmentation needed — MNIST is already clean.
④ Network Implement LeNet5 as nn.Cell with construct() method (Lesson 14). Input: 32×32×1 (MNIST images are padded from 28×28).
⑤ Training Use Adam (lr=0.001), SoftmaxCrossEntropyWithLogits loss, batch size 32, 10 epochs. Watch validation accuracy per epoch via ModelArts dashboard.
⑥ Application Export model to MindIR format. Deploy via MindSpore Serving as a REST API. Mobile client sends image → server returns digit prediction.

4. RNN Input/Output Types

Covered in Lesson 12: RNNs can process sequences. But the input and output don't have to be the same length — this flexibility is what makes RNNs (and LSTMs) useful for many different task types.

Type Input → Output Real-world example
One-to-One Single → Single Image classification (not really sequential, but the degenerate case)
One-to-Many Single → Sequence Image captioning (one photo → a sentence of words)
Many-to-One Sequence → Single Sentiment analysis (a review → positive/negative label)
Many-to-Many Sequence → Sequence Machine translation (sentence → sentence) or POS tagging (word → tag for each word)
⚠ Exam Alert
The exam may describe a task and ask which RNN type applies. Key examples: sentiment analysis = many-to-one, machine translation = many-to-many (often via Seq2Seq), image captioning = one-to-many.
