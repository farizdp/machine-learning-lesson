What Is Machine Learning — and What Can It Solve?

The four types of ML, three problem categories, and why ML beats writing rules by hand.

1. ML vs. Traditional Rule-Based Programming

Before ML existed, programmers solved problems by writing explicit rules. Let's see why that falls short — and what ML does differently.

🔧 Traditional Rule-Based
Input: Data + Hand-written Rules
Output: Answers

A human expert studies the problem, writes IF-THEN rules, and programs them in. The machine just follows those rules.

Works great when: rules are simple and don't change.
Breaks when: rules are too complex or unknown (e.g. "what does a cat look like?").
🧠 Machine Learning
Input: Data + Answers
Output: Rules (learned automatically)

You give the machine examples of inputs and correct outputs. It figures out the rules itself.

Works great when: you have lots of data but can't write the rules.
Breaks when: data is scarce, low quality, or biased.
Analogy
Teaching a child to recognize dogs.
Rule-based: "A dog has four legs, fur, a snout, a tail, ears…" — but a cat also fits! Rules can't capture everything.
ML: Show the child 1,000 photos labelled "dog" and "not dog." They learn what a dog looks like without you ever defining it explicitly.

When Should You Use Machine Learning?

ML is the right tool when at least one of these is true:

  • The problem is too complex to write rules for (face recognition, spam filtering)
  • The rules change constantly and a human can't keep up (stock prices, fraud patterns)
  • You need to personalize for millions of users at once (recommendations, search ranking)
  • You want to discover patterns humans haven't thought to look for (drug discovery, anomaly detection)
⚠ Exam Alert
Know the key difference: Traditional programming = Data + Rules → Answers. Machine Learning = Data + Answers → Rules. The direction of "rules" is flipped.

2. Three Main Problems ML Solves

Almost every ML application falls into one of three categories. Learn to recognize them from a problem description.

📂 Classification — "Which category does this belong to?"
The output is a discrete label from a fixed set of categories.

Examples:
  • Email → spam or not spam (2 categories)
  • Photo → cat, dog, or bird (3 categories)
  • Transaction → fraudulent or legitimate
  • Tumour → malignant or benign
📈 Regression — "What number will this be?"
The output is a continuous numeric value.

Examples:
  • House → predicted sale price ($342,000)
  • Tomorrow → predicted temperature (23.5°C)
  • Ad → predicted click-through rate (3.7%)
  • Patient → predicted hospital stay (4.2 days)
🔵 Clustering — "Which things are similar to each other?"
There are no labels — the machine discovers natural groupings in the data.

Examples:
  • Customers → group by purchasing behaviour (without pre-defined segments)
  • News articles → group by topic automatically
  • Genes → group by expression pattern
  • Documents → group by language or subject
⚠ Exam Alert
The key distinguisher: Classification → discrete label (one of a fixed set). Regression → continuous number. Clustering → no labels, discover groups. A common trick question: "predict tomorrow's temperature" is regression, not classification, because the answer is a number on a continuous scale.

3. Four Types of Machine Learning

ML algorithms are classified by how they learn — specifically, what kind of feedback they get during training.

① Supervised Learning

The training data is labelled — every input comes with the correct answer. The model learns by comparing its predictions to the known correct answers.

Analogy
A student learning from a textbook with an answer key. Every practice problem has a correct answer — the student checks their work and corrects their mistakes.

Used for: Classification and Regression tasks (when you have labelled data).

Examples: Spam detection, house price prediction, image classification, medical diagnosis.

② Unsupervised Learning

The training data is unlabelled — no correct answers given. The model must find its own structure in the data.

Analogy
Sorting a pile of foreign coins you've never seen before. Nobody tells you the categories — you group them yourself by size, colour, and markings. You might make 3 groups or 5; there's no single "correct" answer.

Used for: Clustering, dimensionality reduction, anomaly detection.

Examples: Customer segmentation, topic modelling, fraud anomaly detection, data compression.

③ Semi-Supervised Learning

Uses a small amount of labelled data plus a large amount of unlabelled data. The labelled data guides learning; the unlabelled data helps the model understand the structure.

Analogy
A student who has 10 solved examples and 1,000 unsolved ones. They use the 10 solved problems to understand the pattern, then apply it to make sense of the other 1,000.

Why it matters: Labelling data is expensive and slow (humans have to do it). In the real world, you often have millions of unlabelled data points but only thousands of labelled ones. Semi-supervised learning makes the most of both.

Examples: Medical image analysis (few labelled scans, many unlabelled), web content classification, speech recognition.

④ Reinforcement Learning (RL)

An agent takes actions in an environment, receives rewards for good actions and penalties for bad ones, and learns to maximise cumulative reward over time. There's no labelled dataset — only feedback from the environment.

Analogy
Training a dog with treats. You don't explain the rules — you reward good behaviour and ignore or correct bad behaviour. Over many repetitions, the dog learns the optimal strategy.

Key terms:

Agent
The learner/decision-maker (e.g. the game-playing AI)
Environment
The world the agent interacts with (e.g. the game board)
Reward
A numerical signal indicating how good the last action was
Policy
The strategy the agent follows — what action to take in each situation

Examples: AlphaGo (mastered Go), robot locomotion, game-playing agents (Atari, chess), recommendation systems, autonomous driving.

⚠ Exam Alert
Know all four types and their key traits:
Supervised = labelled data, learns from correct answers
Unsupervised = unlabelled data, finds structure itself
Semi-supervised = small labelled + large unlabelled
Reinforcement = agent + environment + reward signal (no dataset at all)

4. Quick Comparison Table

Type Training Data Feedback Signal Typical Problems
