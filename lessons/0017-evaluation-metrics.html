Model Evaluation Metrics

Accuracy alone can lie. These are the numbers that actually tell you whether your model is good — and good for what.

1. Why Accuracy Can Be Misleading

Suppose you build a cancer screening model. The dataset has 990 healthy patients and 10 cancer patients. A model that predicts "healthy" for every single person gets 99% accuracy — yet it misses every cancer case and is completely useless.

The class imbalance trap
Accuracy = (correct predictions) / (all predictions). When one class vastly outnumbers another, a model can score high accuracy by ignoring the minority class entirely. In medical, fraud detection, and security tasks, the minority class is usually the one that matters most.

You need metrics that distinguish between different types of errors. That starts with the confusion matrix.

2. The Confusion Matrix

For a binary classifier (Positive = disease, Negative = healthy), every prediction falls into one of four cells:

TP — True Positive
Model said Positive, reality is Positive. Correct detection. (Sick patient correctly flagged.)
TN — True Negative
Model said Negative, reality is Negative. Correct rejection. (Healthy patient correctly cleared.)
FP — False Positive (Type I Error)
Model said Positive, reality is Negative. False alarm. (Healthy patient incorrectly flagged — causes unnecessary anxiety and follow-up tests.)
FN — False Negative (Type II Error)
Model said Negative, reality is Positive. Missed detection. (Sick patient incorrectly cleared — often the most dangerous error in medical and security contexts.)
⚠ Exam Alert
Memorise the four cells: TP (correct positive), TN (correct negative), FP (wrong alarm), FN (missed detection). All other metrics are derived from these four numbers.

3. The Core Metrics

Accuracy
(TP + TN) / (TP + TN + FP + FN) — fraction of all predictions that were correct. Only reliable when classes are balanced.
Precision
TP / (TP + FP) — of all the times the model said "Positive", how often was it right? Answers: "When you raise an alarm, how trustworthy is it?" High precision = few false alarms.
Recall (Sensitivity)
TP / (TP + FN) — of all actual Positives, how many did the model catch? Answers: "Of all real cases, how many did you find?" High recall = few missed detections.
F1 Score
2 × (Precision × Recall) / (Precision + Recall) — the harmonic mean of Precision and Recall. Use this as a single number when you care about both. It punishes extreme imbalances between the two (a model with 100% Precision and 0% Recall gets F1 = 0).

Adjusting the classification threshold shifts the Precision–Recall trade-off. F1 balances both.

Worked example — 10 patients

Confusion matrix: TP=8, FN=2, FP=1, TN=89 (100 patients total, 10 sick)

  • Accuracy = (8+89)/100 = 97% — looks great
  • Precision = 8/(8+1) = 88.9% — when we flag someone, we're right 89% of the time
  • Recall = 8/(8+2) = 80% — we caught 8 of the 10 sick patients
  • F1 = 2×(0.889×0.8)/(0.889+0.8) = 84.2%

The 97% accuracy hides that we missed 2 sick patients (FN=2). Recall reveals this directly.

⚠ Exam Alert
Four formulas to memorise:
  • Accuracy = (TP+TN) / (TP+TN+FP+FN)
  • Precision = TP / (TP+FP)
  • Recall = TP / (TP+FN)
  • F1 = 2 × Precision × Recall / (Precision + Recall)

4. ROC Curve and AUC

Precision and Recall depend on the classification threshold you choose. The ROC curve (Receiver Operating Characteristic) shows the model's performance across all possible thresholds at once, so you can evaluate the model independent of any single threshold choice.

The ROC curve plots:

  • X-axis: False Positive Rate (FPR) = FP / (FP + TN) — how often does the model falsely alarm on negatives?
  • Y-axis: True Positive Rate (TPR) = TP / (TP + FN) — this is just Recall

A curve hugging the top-left corner means high TPR at low FPR — a good model. AUC summarises the whole curve as one number.

AUC (Area Under the Curve)
A single number from 0 to 1 summarising the entire ROC curve. Interpretation: the probability that the model ranks a random positive example higher than a random negative example.
AUC = 0.5: no better than random guessing.
AUC = 1.0: perfect classifier.
AUC > 0.9: excellent; 0.7–0.9: acceptable; < 0.7: poor.
⚠ Exam Alert
ROC curve: plots TPR (Y) vs FPR (X) at all thresholds. AUC: area under the ROC curve, ranges 0–1. AUC = 0.5 is random; AUC = 1.0 is perfect. AUC is threshold-independent — it evaluates the model's overall discriminating power regardless of where you set the decision boundary.

5. Choosing the Right Metric

Situation Metric to prioritise Why
Balanced classes, general use Accuracy Fair summary when both classes are equally represented
Missing a positive is costly (cancer, fraud) Recall ↑ Minimise FN (missed real cases)
False alarms are costly (spam filter, legal action) Precision ↑ Minimise FP (false accusations)
Imbalanced classes, both errors matter F1 Score Harmonic mean punishes extreme imbalance between P and R
Comparing models regardless of threshold AUC-ROC Summarises performance across all thresholds in one number

6. Multi-class Extension

The confusion matrix extends naturally to more than two classes — it becomes an N×N grid where cell (i, j) is the count of examples from class i predicted as class j. The diagonal is always correct predictions.

Macro-average
Compute the metric (Precision, Recall, F1) for each class independently, then average them equally. Treats every class as equally important regardless of how many examples it has.
Weighted average
Same as macro, but weights each class's metric by how many examples it has. Better when class sizes differ greatly — large classes contribute more to the final number.
