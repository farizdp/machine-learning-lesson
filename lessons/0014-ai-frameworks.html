AI Development Frameworks

The tools that turn math equations into running code — PyTorch, MindSpore, and how a real model is built.

1. Why Do We Need an AI Framework?

Everything you've learned — gradient descent, backpropagation, CNNs, Transformers — requires implementing thousands of matrix multiplications and computing gradients through complex chains of operations. Doing this from scratch for every model would be impossibly slow and error-prone.

An AI framework provides three things:

Tensor operations
A tensor is the generalisation of a matrix to any number of dimensions. Frameworks provide fast, hardware-accelerated (GPU/NPU) tensor math — add, multiply, reshape, transpose — as simple function calls.
Automatic differentiation (autograd)
The framework tracks every operation you perform and automatically computes the gradient of any output with respect to any input. You never have to derive or code backpropagation by hand.
High-level model building blocks
Pre-built layers (Linear, Conv2d, LSTM), optimisers (Adam), loss functions (CrossEntropyLoss), and dataset utilities — so you can describe what your network is, not how to compute it.

2. Computational Graphs

A computational graph is the internal representation a framework builds to track how tensors flow through operations — so it knows how to compute gradients automatically.

The framework records every operation in the forward pass and uses the chain rule automatically during the backward pass to compute gradients.

Static Graph (Define-and-Run)
The full computation graph is defined first, compiled, and then executed repeatedly with data. Faster at inference, but harder to debug — you can't inspect values mid-graph. Used by TensorFlow 1.x and MindSpore's Graph Mode.
Dynamic Graph (Define-by-Run)
The graph is built on-the-fly as operations execute. You can print tensor values anywhere, use normal Python control flow (if/for), and debug interactively. Used by PyTorch's default mode and MindSpore's PyNative Mode.
⚠ Exam Alert
Static graph = define then run = faster inference, harder to debug. Dynamic graph = define while running = easier to debug, more flexible. MindSpore supports both: Graph Mode (static) and PyNative Mode (dynamic).

3. PyTorch

PyTorch (Meta AI, 2016) is the dominant research framework globally. It uses a dynamic graph by default, which makes it feel like writing ordinary Python.

torch.Tensor
The core data type — an n-dimensional array backed by GPU memory if available. Supports all standard math operations. Every tensor records which operations created it (the gradient tape).
torch.nn.Module
The base class for all models and layers. Override __init__ to define layers and forward() to define the computation. PyTorch automatically generates the backward pass.
torch.optim
Ready-made optimisers: SGD, Adam, AdamW. Call optimizer.step() after loss.backward() to update weights.
torch.utils.data
Dataset and DataLoader — standard interfaces for loading, batching, and shuffling training data.
The PyTorch training loop (4 lines at its core)
optimizer.zero_grad()   # clear old gradients
output = model(inputs)  # forward pass
loss = criterion(output, labels)  # compute loss
loss.backward()         # backpropagation (autograd)
optimizer.step()        # update weights

4. MindSpore — Huawei's AI Framework

MindSpore is Huawei's open-source AI framework, designed to run efficiently on Huawei's Ascend NPU chips as well as CPUs and GPUs. It is the primary framework tested in the HCIA-AI exam.

MindSpore's layered architecture: one API, two execution modes, one IR for optimisation, multiple hardware backends.

mindspore.nn
High-level layer classes — nn.Conv2d, nn.Dense (fully-connected), nn.ReLU, nn.MaxPool2d, nn.SequentialCell. Same concept as PyTorch's torch.nn.
mindspore.ops
Lower-level primitive operations — ops.MatMul, ops.Softmax, ops.GatherV2. Used when you need fine-grained control over the graph.
mindspore.Tensor
MindSpore's tensor type. Created from NumPy arrays or Python lists. Supports .asnumpy() to convert back.
nn.Cell
MindSpore's base class for all models and layers — equivalent to PyTorch's nn.Module. Override __init__ and construct() (not forward()).
Model.train() / Model.eval()
MindSpore's high-level Model class wraps a network, loss function, and optimiser. model.train(epochs, dataset) runs the full training loop automatically.
⚠ Exam Alert
MindSpore key differences from PyTorch: the model base class is nn.Cell (not nn.Module), the forward method is called construct() (not forward()), and MindSpore's two modes are Graph Mode (static, faster) and PyNative Mode (dynamic, easier to debug). MindSpore is optimised for Huawei's Ascend NPU.

5. LeNet — A Classic CNN in Code

LeNet-5 (LeCun, 1998) was the first successful CNN, originally used to recognise handwritten digits (MNIST). It's the "Hello World" of deep learning — small enough to understand completely, real enough to actually work.

LeNet-5: two Conv+Pool pairs for feature extraction, then three FC layers for classification (10 digit classes).

Here is LeNet implemented in MindSpore — the exam-relevant framework. Notice how it matches the architecture diagram directly:

LeNet in MindSpore
import mindspore.nn as nn

class LeNet5(nn.Cell):
    def __init__(self):
        super(LeNet5, self).__init__()
        self.conv1  = nn.Conv2d(1, 6, 5)      # 1 in-channel, 6 filters, 5×5
        self.pool1  = nn.AvgPool2d(2, 2)      # 2×2 average pool, stride 2
        self.conv2  = nn.Conv2d(6, 16, 5)     # 6 → 16 filters
        self.pool2  = nn.AvgPool2d(2, 2)
        self.flatten = nn.Flatten()            # 5×5×16 = 400
        self.fc1    = nn.Dense(400, 120)
        self.fc2    = nn.Dense(120, 84)
        self.fc3    = nn.Dense(84, 10)        # 10 digit classes
        self.relu   = nn.ReLU()

    def construct(self, x):                   # "construct" not "forward"
        x = self.pool1(self.relu(self.conv1(x)))
        x = self.pool2(self.relu(self.conv2(x)))
        x = self.flatten(x)
        x = self.relu(self.fc1(x))
        x = self.relu(self.fc2(x))
        return self.fc3(x)                    # logits (no softmax here)
Training LeNet in MindSpore (high-level API)
from mindspore.train import Model
import mindspore.nn as nn

net       = LeNet5()
loss_fn   = nn.SoftmaxCrossEntropyWithLogits(sparse=True, reduction='mean')
optimizer = nn.Adam(net.trainable_params(), learning_rate=0.001)

model = Model(net, loss_fn=loss_fn, optimizer=optimizer, metrics={"accuracy"})
model.train(epochs=10, train_dataset=train_ds)
model.eval(eval_dataset=test_ds)
⚠ Exam Alert
Know the LeNet-5 layer sequence: Conv(6) → Pool → Conv(16) → Pool → Flatten → FC(120) → FC(84) → FC(10). Both pooling layers use average pooling (not max pooling) in the original LeNet. In MindSpore the forward method is construct(), not forward().
