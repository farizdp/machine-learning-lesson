Convolutional Neural Networks (CNN)

How AI learned to see — the architecture behind every image recognition system.

1. Why Not Just Use a Fully-Connected Network for Images?

In Lesson 08 you learned about fully-connected (FC) networks where every neuron connects to every other. For images, this breaks down immediately:

The scale problem
A small 224×224 colour image has 224 × 224 × 3 = 150,528 input values. If the first hidden layer has 1,000 neurons, that's already 150 million parameters — just for layer one. Untrainable, and the network ignores the spatial structure of the image entirely.

CNNs solve both problems by exploiting two facts about images:

  • Local patterns matter — an edge or texture is defined by nearby pixels, not pixels across the image
  • The same pattern appears anywhere — a "cat ear" detector should work in the top-left and bottom-right corner equally
Analogy
Instead of asking "how does every pixel relate to every other pixel?", a CNN slides a small magnifying glass (the kernel) across the image and looks for local patterns — edges, curves, textures. The same glass is used everywhere, which massively reduces the number of parameters needed.

2. The Convolution Operation

The core of a CNN is the convolution: slide a small grid called a kernel (or filter) across the input, compute a dot product at each position, and record the results in an output feature map.

Kernel (Filter)
A small grid of learnable weights (e.g. 3×3, 5×5). Each kernel learns to detect a specific pattern — one might detect horizontal edges, another vertical edges, another curves. A single convolutional layer uses many kernels in parallel.
Stride
How many pixels the kernel moves at each step. Stride=1: move one pixel at a time (produces a large output). Stride=2: skip every other position (halves the output size).
Padding
Adding a border of zeros around the input before convolving. Same padding: output is the same size as input. Valid padding: no border added, output shrinks.
Feature Map
The output of applying one kernel to the entire input. If you have 32 kernels, you get 32 feature maps — one per kernel — stacked into a volume.
⚠ Exam Alert
Output size formula for one dimension: (Input − Kernel + 2×Padding) / Stride + 1. For a 5×5 input, 3×3 kernel, stride 1, no padding: (5 − 3 + 0)/1 + 1 = 3. This matches the diagram above.

4. Convolutional Layer

The convolutional layer applies multiple kernels to the input simultaneously. Each kernel produces one feature map; together they form a 3D output volume.

Parameter sharing
Every position in the input shares the same kernel weights. A 3×3 kernel applied to any size image has only 9 learnable weights — regardless of image size. This is why CNNs have vastly fewer parameters than FC networks.
Local connectivity
Each output neuron only "sees" a small local region of the input (the receptive field of the kernel). Irrelevant distant pixels don't interfere.
What kernels learn
Early layers: low-level features (edges, corners, colour gradients). Middle layers: textures and shapes. Deep layers: high-level parts (eyes, wheels, words).
⚠ Exam Alert
Two key properties of convolutional layers: parameter sharing (same kernel weights used everywhere) and local connectivity (each neuron only sees a local region). These are what make CNNs efficient for images.

5. Pooling Layer

Pooling downsamples the feature maps — it reduces the spatial size while keeping the most important information. This reduces computation and builds some translation invariance (the same feature detected slightly shifted still gives the same pooled output).

Max Pooling (2×2, stride 2)
Take the maximum value in each 2×2 region
Average Pooling (2×2, stride 2)
Takes the average of each 2×2 region instead of the maximum.

Both reduce spatial size by 2× in each dimension when stride = pool size.
Max Pooling
Takes the largest value in each region. Keeps the strongest feature response. Most commonly used. Helps the network be robust to small translations — if the feature shifts one pixel, the max in the region stays the same.
Average Pooling
Takes the mean of each region. Smoother, retains more context. Used in some architectures (e.g. Global Average Pooling at the end of modern CNNs instead of a Flatten layer).
No learnable parameters
Pooling layers have no weights — they just downsample. This makes them cheap and they don't contribute to overfitting.
⚠ Exam Alert
Max Pooling = take the maximum in each window (most common). Average Pooling = take the mean. Both reduce spatial dimensions without learnable parameters. Pooling adds translation invariance.

6. Fully-Connected Layer & the Full Picture

After the convolutional and pooling layers have extracted spatial features, the output is flattened into a 1D vector and fed into standard fully-connected layers for classification.

Flatten
Convert the 3D feature volume (height × width × channels) into a single 1D vector. No computation — just reshaping.
Fully-Connected (Dense) Layer
One or more FC layers combine the extracted features into a final prediction. Last layer uses Softmax for multi-class output.
The full CNN pipeline in one sentence
Convolutional layers act as a feature extractor (learning what's in the image), and fully-connected layers act as a classifier (deciding which category based on those features).
⚠ Exam Alert
Know the three CNN layer types and their roles: Convolutional (extract features, has learnable kernels), Pooling (downsample, no learnable parameters), Fully-Connected (classify based on extracted features). The typical order is: Conv → ReLU → Pool → … → Flatten → FC → Softmax.
