1. Linear Regression

Task: Regression — predict a continuous number.

Linear regression finds the best straight line through your data points. Given input features (like house size), it predicts a numeric output (like house price).

Red dashed = error (residual). Training minimizes the sum of squared errors.

The formula
ŷ = w₁x₁ + w₂x₂ + … + b — the model learns the weights (w) and bias (b). These are the parameters.
When to use
When you expect a roughly linear relationship between inputs and output. Fast, interpretable, and a great baseline.
Limitation
Only captures straight-line relationships. Real-world data is often curved → need Polynomial Regression.

2. Polynomial Regression

Task: Regression — predict a continuous number (with curves).

Polynomial Regression extends linear regression by adding squared, cubed, etc. versions of the features. This lets the model fit curves instead of just straight lines — but it's still a linear model under the hood (linear in its parameters).

Polynomial regression fits curved relationships by adding x², x³… as extra features.

⚠ Exam Alert
Linear Regression = straight line. Polynomial Regression = curved line. Higher-degree polynomials fit more complex curves but risk overfitting on small datasets.

3. Logistic Regression

Task: Classification — predict a category (despite the name "regression").

Despite its name, Logistic Regression is a classification algorithm. It predicts the probability that something belongs to a class. The output is always between 0 and 1.

Analogy
You're predicting whether an email is spam (1) or not spam (0). Logistic Regression gives you "there's an 87% chance this is spam" — then you apply a threshold (e.g. > 50% = spam) to make the final decision.

The key ingredient is the sigmoid function which squashes any number into the 0–1 range:

Sigmoid squashes all inputs into (0, 1) — readable as a probability.

⚠ Exam Alert
Logistic Regression solves classification, not regression, despite its name. It uses the sigmoid function to output a probability between 0 and 1. Common use: binary classification (spam/not-spam, yes/no).

4. Decision Tree

Task: Both Classification and Regression.

A Decision Tree learns a series of if/else questions about the features and splits the data step by step. At each node, it asks the question that best separates the classes.

Pros
Highly interpretable — you can read the tree like a flowchart. No need to scale features.
Cons
Prone to overfitting — trees can grow very deep and memorize training data. Solution: limit depth or use ensemble methods (see Random Forest below).

5. Support Vector Machine (SVM)

Task: Classification (and Regression).

SVM finds the decision boundary that maximizes the margin — the gap between classes. It's not just any boundary; it's the one with the widest "no-man's-land" between the classes.

SVM finds the widest possible gap between classes. Points closest to the boundary are "support vectors."

Linear SVM
Draws a straight-line (or flat hyperplane) boundary between classes. Works when data is linearly separable — the two classes can be separated by a straight line.
Non-Linear SVM (Kernel Trick)
When a straight line can't separate the classes, SVM uses a kernel function to project the data into a higher dimension where a flat boundary does work. Common kernels: RBF (Radial Basis Function), Polynomial.
⚠ Exam Alert
SVM maximizes the margin between classes. Points closest to the boundary are called support vectors. Use kernel trick for non-linearly separable data. Know the difference between linear SVM and kernel SVM.

6. k-Nearest Neighbors (k-NN)

Task: Classification and Regression.

k-NN is the simplest idea in ML: to classify a new point, look at the k closest points in the training data and take a vote. The new point gets the majority class.

With k=3, 2 neighbors are Class A and 1 is Class B → new point classified as Class A.

The k hyperparameter
Small k (e.g. k=1) → very sensitive to noise, overfits. Large k → smoother boundary, but may miss local patterns. Tune k with cross-validation.
Distance measures
The most common is Euclidean distance (straight-line distance). You must scale your features before using k-NN — otherwise large-scale features dominate the distance calculation.
Lazy learner
k-NN has no real "training phase" — it just memorizes all the data. Prediction is slow because it must compute distances to every training point.

7. Naive Bayes

Task: Classification.

Naive Bayes applies Bayes' theorem — a probability rule — to classify data. It calculates the probability of each class given the features, and picks the highest.

The "naive" part: it assumes all features are independent of each other. In reality this is rarely true, but the algorithm still works surprisingly well in practice.

Analogy — Spam Filter
You want to classify an email as spam. Naive Bayes says: "Given that this email contains the words 'free', 'win', and 'prize', what's the probability it's spam?" It multiplies the individual word probabilities together (the "naive" independence assumption), then picks the class with the highest probability. This is exactly how classic email spam filters work.
When it shines
Text classification (spam detection, sentiment analysis, document categorization). Very fast to train and works well with small datasets and high-dimensional data.
Core assumption (the "naive" part)
Features are conditionally independent given the class. This is almost never true in the real world, but the algorithm is robust enough that it works well anyway.
⚠ Exam Alert
Naive Bayes is a probabilistic classifier based on Bayes' theorem. The "naive" assumption is feature independence. Classic use case: text classification / spam filtering.

8. Ensemble Learning — Random Forest & GBDT

Task: Classification and Regression.

The idea behind ensemble learning: combine many weak models to build one strong model. Individually, each model might make mistakes — but together, their errors cancel out.

votes: A
↓ majority vote
Final answer: Class A (4/5 votes)

There are two main strategies for building ensembles:

Bagging (Bootstrap Aggregating)
Train many models in parallel, each on a random subset of the training data. Combine their predictions by voting (classification) or averaging (regression). Each model is independent.

Goal: reduces variance (overfitting). Models are diverse because they saw different data.
Boosting
Train models sequentially. Each new model focuses on the mistakes the previous models made. Final answer is a weighted combination of all models.

Goal: reduces bias (underfitting). Later models correct earlier models' errors.

Random Forest (Bagging-based)

Random Forest trains many Decision Trees using bagging, with one extra twist: each tree also only sees a random subset of features at each split. This makes trees less correlated, which improves accuracy.

Why it beats a single Decision Tree
A single tree overfits easily. Many diverse trees, each slightly wrong in different ways, average out their errors and produce a much more stable, accurate result.
Key hyperparameters
Number of trees (n_estimators), max depth of each tree, number of features considered at each split.

Gradient Boosted Decision Tree (GBDT) — Boosting-based

GBDT trains trees sequentially. Each new tree learns to predict the residual errors (the mistakes) of the previous ensemble. Famous implementations: XGBoost, LightGBM, CatBoost.

Why it's powerful
GBDT consistently wins structured/tabular data competitions. It handles mixed feature types, missing values, and non-linear patterns extremely well.
Risk
More prone to overfitting than Random Forest. Requires careful hyperparameter tuning (learning rate, number of trees, max depth).
⚠ Exam Alert
Bagging = parallel independent models, reduces variance. Boosting = sequential corrective models, reduces bias. Random Forest = bagging + random feature selection. GBDT = boosting with gradient descent on residuals.

9. k-Means Clustering

Task: Unsupervised — find natural groups with no labels.

k-Means divides data into k clusters by iteratively assigning each point to its nearest cluster center (centroid) and re-computing the centroids.

Step 1: Init
Place k centroids randomly
Step 2: Assign
Assign each point to nearest centroid
Step 3: Update
Move centroids to cluster mean
Step 4: Repeat
Repeat until centroids stop moving
You must choose k
The number of clusters k is a hyperparameter you set in advance. Use the Elbow Method: plot the total within-cluster variance for different k values — pick k at the "elbow" where adding more clusters gives diminishing returns.
Limitation
Sensitive to initial centroid placement. Results can vary. Run multiple times and pick the best. Also assumes clusters are roughly spherical and equal-sized.
⚠ Exam Alert
k-Means is unsupervised. The algorithm alternates between assigning points to nearest centroids and updating centroids to cluster means. You must choose k in advance. Use the Elbow Method to find a good k.

10. Hierarchical Clustering

Task: Unsupervised — find groups without specifying k in advance.

Hierarchical Clustering builds a tree of clusters (dendrogram). Unlike k-Means, you don't need to choose the number of clusters beforehand.

Agglomerative (bottom-up) — most common
Start with every point as its own cluster. Repeatedly merge the two closest clusters until only one big cluster remains. Cut the dendrogram at the desired level to get k clusters.
Divisive (top-down)
Start with one big cluster containing all points. Repeatedly split until each point is its own cluster.

Dendrogram: cutting at different heights gives different numbers of clusters. The yellow line cuts for 3 clusters.

Advantage over k-Means
You don't need to choose k in advance — you can cut the dendrogram at any level. Also shows the full structure of how clusters relate to each other.
Disadvantage
Computationally expensive for large datasets — doesn't scale well to millions of points.
